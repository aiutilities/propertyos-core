# PropertyOS Constitution

## Mission

Become the default open-source platform for managing people, spaces, visitors, assets, and operations across physical properties worldwide.

## Principles

1. Open Source First
2. Data Ownership
3. Plugin Driven Architecture
4. API First
5. Workflow First
6. AI Assisted Administration
7. Mobile First
8. WhatsApp First for Emerging Markets
9. Self Host or Cloud Host
10. Marketplace Ready

## Platform Rules

- PropertyOS Core owns shared infrastructure.
- Plugins own business-specific functionality.
- Themes own presentation and experience.
- Workflows own automation.
- Core must remain property-type neutral.
- Admins must be able to install plugins and themes from the admin panel.
- No server login should be required for normal plugin or theme management.
- PropertyOS must support both self-hosted and cloud-hosted deployment.
- PropertyOS must allow different solutions to be built on top of the same open core.

## Non Goals

- Not an apartment management system only
- Not a visitor management system only
- Not an accounting package
- Not an ERP
- Not a CRM
- Not a closed SaaS-only product
