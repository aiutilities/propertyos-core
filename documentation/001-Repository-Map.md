# PropertyOS Repository Map

## Purpose

This document explains the purpose of every top-level folder.

---

## constitution

Mission, principles, vision, and platform rules.

---

## architecture

Domain model, architecture, core modules, and technical design.

---

## adrs

Architecture Decision Records.

Permanent record of major decisions.

---

## prd

Product requirements and product strategy.

---

## sdk

Developer extension specifications.

Plugins, themes, workflows, and future extension models.

---

## backend

Core platform backend implementation.

Contains:

* Identity
* Organization
* Property
* People
* Workflow
* Event Bus
* Notification
* Plugin
* Theme
* AI Layer

---

## frontend

User interface implementation.

Contains:

* Layouts
* Pages
* Components
* Themes
* Plugin UI

---

## plugins

Official PropertyOS plugins.

Examples:

* Visitor Management
* Asset Management
* Helpdesk
* Booking

---

## themes

Official PropertyOS themes.

---

## distributions

Packaged solutions built on PropertyOS.

Examples:

* CommunityOS
* CommercialOS
* CoworkOS

---

## infrastructure

Deployment and hosting configuration.

---

## documentation

User, administrator, developer, and operational documentation.

---

## Goal

Every contributor should be able to understand the repository structure within minutes.
